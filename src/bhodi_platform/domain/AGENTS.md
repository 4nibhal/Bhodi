# Rules: Domain Layer

## Context & Responsibility
Pure business logic with zero external dependencies. This layer contains entities, value objects, domain services, and domain exceptions. Nothing here knows about adapters, infrastructure, or transport layers.

## Architectural Contract
- **Zero imports** from other bhodi_platform layers (no imports from `application/`, `ports/`, `infrastructure/`, `interfaces/`)
- **Zero external dependencies** (no imports from `langchain`, `chromadb`, `fastapi`, etc.)
- All dependencies are passed in via constructor or as arguments
- Entities are the source of truth for business invariants

## Domain Components

### Entities (`domain/entities.py`)
- `Document`: Represents an indexed document with metadata
- `Chunk`: A fragment of a document with position information
- `Query`: A user query with optional session context
- `Answer`: A generated response with citations
- `ConversationTurn`: A single exchange in a conversation

### Value Objects (`domain/value_objects.py`)
- `DocumentId`: Unique identifier for a document
- `ChunkId`: Unique identifier for a chunk
- `EmbeddingVector`: An embedding as an immutable sequence of floats
- `Citation`: A reference to a source chunk with position info
- `ConversationId`: Unique identifier for a conversation session

### Domain Services (`domain/services.py`)
- `ChunkingService`: Pure chunking logic (no I/O)
- `RetrievalService`: Pure retrieval ranking logic (no I/O)

### Domain Exceptions (`domain/exceptions.py`)
- `DocumentNotFoundError`: Raised when a document doesn't exist
- `ChunkingError`: Raised when chunking fails
- `EmbeddingError`: Raised when embedding generation fails
- `VectorStoreError`: Raised for vector storage failures
- `InvalidDocumentError`: Raised when document format is invalid

## Quality Standards
- All classes must have complete type hints (no `Any`)
- All public methods must have docstrings
- Value objects must be immutable (frozen dataclasses)
- Entities must validate invariants in `__post_init__`
- Exceptions must be descriptive with context

## Testing Contract
- Domain tests must have zero external dependencies
- All business rules must be testable without mocks
- Use pure Python assertions, no pytest helpers needed

## Delegation & Boundaries
- Inheritance: Inherits from `src/bhodi_platform/`
- This layer is **never** imported by infrastructure or interfaces directly
- Application layer orchestrates domain entities
