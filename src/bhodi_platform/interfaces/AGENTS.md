# Rules: Interfaces Layer

## Context & Responsibility
Transport adapters that expose bhodi_platform functionality to the outside world. This layer contains API servers (FastAPI), CLI commands (Typer), and worker queue adapters. No business logic here—only request handling, response serialization, and transport-specific concerns.

## Architectural Contract
- All transport-specific code lives here
- Request handlers call application layer (BhodiApplication facade)
- Responses are serialized from domain models
- Errors mapped to HTTP status codes
- No business logic or domain rules

## API Server (FastAPI)

### Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/health` | Health check (no model init) |
| POST | `/documents` | Index a document |
| DELETE | `/documents/{id}` | Remove a document |
| POST | `/query` | Query the index |
| GET | `/conversations/{id}` | Get conversation history |

### Error Responses
All errors follow HTTP conventions:

| Status | When |
|--------|------|
| 400 | Invalid request (malformed body, invalid params) |
| 404 | Document not found |
| 422 | Validation error (Pydantic) |
| 500 | Internal error (wrapped domain exception) |
| 503 | Service unavailable (external API down) |

### Middleware
- `ErrorHandlingMiddleware`: Maps domain exceptions to HTTP responses
- `CORSMiddleware`: Configurable CORS
- `RequestLoggingMiddleware`: Logs requests with correlation ID

## CLI Commands (Typer)

Commands mirror API functionality:

```bash
# Index documents
bhodi-index upload ./docs/*.pdf --metadata source=legal

# Query
bhodi query "What is the termination clause?" --conversation-id abc123

# Health check
bhodi health
```

## Worker Adapter

For background job processing:
- Accepts tasks from queue (Celery, Redis, etc.)
- Delegates to `BhodiApplication`
- Returns results or errors to queue

## Request/Response Models

API requests use Pydantic models from `application/models.py`:
- No duplicate models in interfaces layer
- FastAPI handles validation automatically

## Quality Standards
- All endpoints have OpenAPI documentation
- All CLI commands have help text
- Error messages are user-friendly (no stack traces in responses)
- Requests are validated before passing to application

## Telemetry
- HTTP requests traced via middleware
- CLI invocations traced at entry point
- Correlation ID propagated through call stack

## Testing Contract

- CLI tests: Use Typer's CliRunner
- Mock BhodiApplication in all interface tests
