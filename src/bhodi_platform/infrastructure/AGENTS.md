# Rules: Infrastructure Layer

## Context & Responsibility
Concrete adapter implementations that fulfill port contracts. This layer contains all external integrations (OpenAI, Chroma, PyPDF, etc.) and the dependency injection container that wires everything together.

## Architectural Contract
- Each adapter lives in its own subdirectory under `infrastructure/`
- Each adapter implements exactly one port
- Adapters are instantiated via factory functions or Container
- No global state or singletons (except where necessary for performance)
- Lazy initialization: models loaded on first use, not at import time

## Container (`composition.py`)

The Container is the main entry point for wiring:

```python
class Container:
    @classmethod
    def from_config(cls, config: BhodiConfig) -> Container: ...
    
    def build(self) -> BhodiApplication: ...
    
    def get_port(self, port_type: type[Protocol]) -> Any: ...
```

### Wiring Rules
1. Config determines which adapter implementation to use
2. Each adapter is instantiated lazily
3. Health check never triggers adapter initialization
4. Container is the only place that knows about concrete types

## Adapter Standards

### Initialization
```python
class OpenAIEmbeddings:
    def __init__(self, config: EmbeddingConfig) -> None:
        self._config = config
        self._client = None  # Lazy
    
    async def _ensure_client(self) -> None:
        if self._client is None:
            self._client = openai.AsyncClient(...)  # Created on first use
```

### Error Handling
- Wrap external exceptions in domain exceptions
- Include context (provider, model, operation) in error messages
- Log errors with appropriate severity

### Telemetry
- Each adapter adds span attributes: `provider`, `model`, `operation`
- Duration measured from call start to completion

## Quality Standards
- All adapters must pass contract tests
- Mock adapters must be deterministic (same input = same output)
- No API keys stored in code (use environment variables)
- All config via `config: SomeConfig` constructor arg

## Testing
- Integration tests use real adapters with local services (embedded Chroma)
- E2E tests use mock adapters (no external API calls)
- Each adapter has a corresponding contract test

## Delegation & Boundaries
- Inheritance: Inherits from `src/bhodi_platform/`
- Implements ports defined in `ports/`
- Instantiated and wired by `application/` via Container
