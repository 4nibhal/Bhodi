# Rules: Application Layer

## Context & Responsibility
Use cases and orchestration logic. This layer sits between domain and infrastructure, composing ports to implement business workflows. It knows about ports (interfaces) but not about concrete adapters.

## Architectural Contract
- Imports domain entities and port protocols only
- Does NOT import concrete adapters from `infrastructure/`
- Contains request/response models (Pydantic) for API contracts
- Contains configuration models (Pydantic) for runtime config
- All external I/O goes through ports (dependency injection)

## Application Components

### Configuration (`application/config.py`)
- `EmbeddingConfig`: Embedding provider settings
- `VectorStoreConfig`: Vector store provider settings
- `LLMConfig`: LLM provider settings
- `ChunkerConfig`: Chunker provider settings
- `DocumentParserConfig`: Parser provider settings
- `ConversationConfig`: Conversation memory settings
- `BhodiConfig`: Root configuration aggregating all sub-configs

### Use Cases (`application/*.py`)
- `IndexDocumentUseCase`: Orchestrates document indexing pipeline
- `QueryAnswerUseCase`: Orchestrates query-answering pipeline
- `BhodiApplication` (facade): Main entry point combining all use cases

### Request/Response Models (`application/models.py`)
- `IndexDocumentRequest`: Input for indexing
- `IndexDocumentResponse`: Output from indexing
- `QueryRequest`: Input for querying
- `QueryResponse`: Output from querying
- `HealthStatus`: Health check response

## Quality Standards
- All config models use Pydantic with Field constraints
- No hardcoded values (models, temperatures, chunk sizes, paths)
- Provider-specific config via `extra: dict[str, Any]`
- Use async/await for all port calls
- Include OpenTelemetry spans for all operations

## Error Handling
- Domain exceptions propagate up unchanged
- Infrastructure exceptions wrapped in domain exceptions with context
- HTTP status codes mapped in interfaces layer (not here)

## Telemetry
- Spans for: `indexing.parse`, `indexing.chunk`, `indexing.embed`, `indexing.store`
- Spans for: `query.embed`, `query.search`, `query.generate`
- Attributes: `document_id`, `query_id`, `provider`, `duration_ms`

## Testing Contract
- Use mocks for ports (not real adapters)
- Test orchestration logic, not external services
- Include happy path and error path tests

## Delegation & Boundaries
- Inheritance: Inherits from `src/bhodi_platform/`
- Calls ports (protocols), not concrete adapters
- Instantiated by infrastructure composition container
